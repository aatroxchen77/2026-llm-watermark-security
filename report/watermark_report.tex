\begin{abstract}
Kirchenbauer 等人提出的红绿名单 Logit 偏置水印方案是大模型内容溯源领域被广泛引用的工作之一。本文在 OPT-1.3B 模型上对该方案进行了独立复现，并在原有生成-检测-攻击链路的基础上补充了四项统计检验：绿名单归属序列的自相关分析、水印与自然分布的 TVD/KL 距离度量、高偏置 token 识别，以及基于偏置先验的定向替换实验。实验数据表明：(1) 水印文本的绿名单归属不存在显著序列相关（$r_1 \approx 0.009$），Z 检验的 IID 假设在长文本条件下具备经验支撑；(2) 水印分布与自然分布之间存在可量化的统计偏移（$\text{TVD} \approx 0.281$），偏移集中在少数高频功能词上；(3) 利用该偏移构造的定向替换攻击效果有限——在 $10\%$ 替换率下 Z-score 从 $10.04$ 降至 $8.44$，仍远高于检测阈值 $\tau=4.0$，提升替换率后面临收益骤降的平台效应。本文的结论是：Kirchenbauer 方案在检测鲁棒性层面表现坚实，基于词频的简单定向攻击难以构成实质性威胁；但词表分布层面的统计偏移确凿存在，是后续形式化安全分析中需要正视的经验事实。
\end{abstract}

\section{引言}
区分大语言模型生成文本与人类写作，对于学术诚信维护、虚假信息治理和知识产权保护都有实际意义。Kirchenbauer 等人提出的红绿名单 Logit 偏置水印方案是该方向被讨论最多的工作之一：生成时对伪随机划分的"绿名单"token 施加正向 logit 偏置，检测端仅需验证绿名单出现频率偏离期望值的程度。其原理简洁、代码公开，适合作为深入检验的对象。

本文定位为一次诚实复现与统计检验，不提出新算法，也不声称理论创新。追问这样一个问题：该方案高度依赖统计假设（token 归属独立、水印分布与自然分布在统计上不可区分），这些假设在多大程度上经得起数据检验？我们在原始代码基础上补充了四项分析——自相关检验、TVD/KL 分布距离、高偏置 token 列表和定向替换实验——让数据直接回答，检验已有结论的稳健性，记录实验中观察到的现象。

\section{相关工作}
大模型文本检测技术可大致分为被动检测和主动水印两类。被动检测不修改生成过程，通过对输出文本的统计特征进行后验溯源。Mitchell 等人提出的 DetectGPT~\cite{mitchell2023detectgpt} 利用了模型 Logits 在生成文本附近的局部曲率异常，实现零样本检测，但计算开销大，且对释义重写较为脆弱。主动水印方案则在生成阶段嵌入可控信号。Kirchenbauer 等人~\cite{kirchenbauer2023watermark,kirchenbauer2023reliability} 提出的红绿名单方案使用 $\gamma=0.25$、$\delta=2.0$ 的参数组合，已被后续工作广泛复现。Wu 等人~\cite{wu2023unbiased} 进一步提出无偏变体以降低质量损失；Zhao 等人~\cite{zhao2024gamma} 系统量化了绿名单比例对检测性能的影响。

在安全分析层面，Aaronson~\cite{aaronson2023watermark} 将水印检测形式化为假设检验问题，Christ 等人~\cite{christ2023undetectable} 从计算复杂性角度给出了不存在多项式时间攻击者能完全消除水印信号的理论边界。在统计泄漏的量化工具方面，Fernandez 等人~\cite{fernandez2023stable} 在扩散模型水印中引入全变差距离。本文将 TVD 分析推广到自回归语言模型场景，作为检验 Kirchenbauer 方案分布特性的经验工具之一。

\section{背景：红绿名单水印方案}

\subsection{水印植入}
\begin{enumerate}
    \item \textbf{上下文哈希}：利用密钥 $K$ 对前 $h$ 个 token 进行哈希，$h_t = \text{SHA256}(x_{t-1} | K)$。
    \item \textbf{词表划分}：将词表 $V$ 确定性地划分为绿名单 $G$（比例 $\gamma$）和红名单 $R$。
    \item \textbf{Logit 偏置}：
    \begin{equation}
        l'_t = l_t + \delta \cdot \mathbf{1}_{G}
    \end{equation}
\end{enumerate}

\subsection{统计检测}
检测端仅需密钥 $K$，无需访问原始文本：
\begin{equation}
    Z = \frac{|s|_G - \gamma T}{\sqrt{T \gamma (1-\gamma)}}
\end{equation}
该统计量在零假设下渐近服从 $N(0, 1)$。实验中使用阈值 $\tau = 4.0$，对应误报率约 $3 \times 10^{-5}$。

\subsection{IID 假设的经验检验}
Z 检验的有效性依赖于各 token 红绿归属为独立同分布伯努利试验的假设。但在自回归生成过程中，token 选择同时受伪随机划分和历史语义的双重约束——相邻 token 的绿名单归属可能存在序列相关。为检验这一点，我们从 10 组生成文本中提取了逐 token 的绿名单归属序列（1 = 绿，0 = 红），计算了 Lag-1 自相关系数。在平均长度 205 tokens 的文本上，$r_1$ 均值为 0.0092（标准差 0.0606），$90\%$ 的样本满足 $|r_1| < 0.1$，有效样本量 $N_{\text{eff}} \approx 202$ 与名义 $T=205$ 基本一致。这表明在当前参数配置下，IID 近似具备充分的经验支持，与原始论文隐含的立场相符。

\section{复现实验}

\subsection{实验设置}
\begin{itemize}
    \item \textbf{生成模型}：\texttt{facebook/opt-1.3b}。
    \item \textbf{攻击模型}：\texttt{Vamsi/T5\_Paraphrase\_Paws}（释义重写）。
    \item \textbf{评估模型}：\texttt{gpt2-large}，用于计算 PPL。使用不同于生成模型的 PPL 评估器可以避免自我评估偏差；GPT-2 系列词表与 OPT 系列同属 BPE 体系，跨模型评估的 PPL 数值具有可比较的语义含义，但绝对值不宜与同分布 PPL 等量齐观。
    \item \textbf{水印参数}：$\gamma=0.25$，$\delta=2.0$。
    \item \textbf{样本}：10 组多样化 Prompt（高熵 5 组、低熵 5 组），每组生成约 200 tokens。
    \item \textbf{硬件}：NVIDIA RTX 3090 (24GB)；统计检验脚本可纯 CPU 运行。
    \item \textbf{不确定性估计}：PPL 与 Z-score 均报告 10,000 次 Bootstrap 的 95\% 置信区间。
\end{itemize}

\section{统计安全性检验}

\subsection{分布偏差：TVD 与 KL 散度}
从统计不可区分性的经验视角出发，我们量化了水印分布 $P_w$ 与自然分布 $P_n$ 的偏离程度。

全变差距离基于 OPT-1.3B 的 50,265 维词表计算。$P_w(v)$ 为 token $v$ 在水印文本中的出现频率（加一平滑，$\alpha=1$），$P_n(v)$ 为基线生成文本中的 token 频率——这一定义将水印引起的偏置与生成模型本身的 token 偏好的耦合效应一并纳入，因此 TVD 反映的是水印文本分布相对于基线文本分布的整体偏移，而非纯粹的"水印注入偏置"：
\begin{equation}
    \text{TVD}(P_w, P_n) = \frac{1}{2} \sum_{v \in V} |P_w(v) - P_n(v)| \approx 0.2809 \, [\text{Bootstrap 95\% CI: } 0.304, 0.403]
\end{equation}

KL 散度提供信息论角度的补充度量：
\begin{equation}
    D_{\text{KL}}(P_w \| P_n) = \sum_{v \in V} P_w(v) \log \frac{P_w(v)}{P_n(v)} \approx 0.251
\end{equation}

要理解哪些 token 主导了这种分布偏移，表~\ref{tab:leakage} 列出了偏置增量 $\Delta(v) = |P_w(v) - P_n(v)|$ 前五高的 token。

\begin{table}[ht]
    \centering
    \caption{分布偏置最显著的 Token}
    \label{tab:leakage}
    \begin{tabular}{lcc}
        \toprule
        Token & $\Delta(v)$ & 偏置程度 \\
        \midrule
        \texttt{\textbackslash n} (换行符) & $7.70 \times 10^{-3}$ & 高 \\
        \texttt{in} & $5.04 \times 10^{-3}$ & 高 \\
        \texttt{on} & $3.85 \times 10^{-3}$ & 中 \\
        \texttt{<unk>} & $3.56 \times 10^{-3}$ & 中 \\
        \texttt{The} & $3.56 \times 10^{-3}$ & 中 \\
        \bottomrule
    \end{tabular}
\end{table}

以上分析确认了水印在词表层面引入了可量化的统计偏置。在解释这些数字时需要谨慎：第一，边际分布的 TVD 不等同于序列级的 IND 区分优势；第二，偏置集中在少数高频功能词上，大多数 token 的 $\Delta(v)$ 接近零；第三，基于这种偏置信息能构造多大的实际区分优势，还取决于攻击者掌握的样本量和计算资源。本文的 TVD/KL 分析是经验层面的统计描述，不构成形式化的 IND 安全性论证。

\subsection{T5 释义攻击的简单线性拟合}
将 T5 释义攻击建模为：攻击者以翻转概率 $p$ 将绿名单 token 改写为红名单 token。攻击后绿名单 token 的期望数为：
\begin{equation}
    \mathbb{E}[|s|_G^{\text{att}}] \approx (1-p) \cdot |s|_G + p \cdot \gamma T
\end{equation}
其中 $p\gamma T$ 为盲写中偶然命中绿名单的期望值。代入 Z 统计量定义可得：
\begin{equation}
    \mathbb{E}[Z_{\text{att}}] \approx (1-2p) \cdot Z_{\text{orig}}
\end{equation}
由群体均值 $Z_{\text{orig}} = 10.04$、$Z_{\text{att}} = 5.24$ 反解 $p \approx 0.239$。逐样本计算的 Z-score 留存率 $R = Z_{\text{att}} / Z_{\text{orig}}$ 的均值为 52.7\%。这个线性关系在群体层面拟合较好，但需要说明它假设翻转概率对所有 token 均一——在 T5 语义保持的约束下，这一假设显然不精确。

\subsection{定向替换实验：统计先验能带来多大攻击优势？}
TVD 分析识别出了一组高偏置 token。很自然地问：如果攻击者知道这份 token 黑名单——Jovanovi\'{c} 等人~\cite{jovanovic2024watermark} 最近的工作表明通过查询访问即可提取水印模式，使得这一假设有现实基础——定向替换的效率是否高于无差别的随机替换？

在两种策略（定向替换优先攻击偏置最大的 token、随机替换将任意位置的 token 替换为常见功能词）下，以 2\% 为步长扫描了 0\%—20\% 的替换率。结果汇总在图~\ref{fig:sensitivity}。

\begin{figure}[ht]
    \centering
    \includegraphics[width=0.85\textwidth]{rational_sensitivity.pdf}
    \caption{定向替换 vs 随机替换的敏感性曲线。低替换率区间（$<8\%$）定向替换略优，体现了统计先验的局部价值；$>10\%$ 后定向替换迅速进入平台期（高偏置 token 已被耗尽），反被随机替换超越。}
    \label{fig:sensitivity}
\end{figure}

实验揭示了三个现象：

\begin{enumerate}
    \item \textbf{定向攻击的作用范围有限}。在 10\% 替换率下，定向替换将 Z-score 降至 8.44，与随机替换的 7.96 相差不大。继续提升替换率，Z-score 稳定在 8.39 左右不再下降——高偏置 token 已被全部替换，剩余 token 的偏置量微乎其微。
    \item \textbf{随机替换在高替换率区间反而更有效}。在 20\% 替换率下，随机替换将 Z 压至 6.68（降幅 33.5\%），明显低于定向替换的 8.39（降幅 16.4\%）。当降噪面足够大时，无差别噪声比弹药有限的高精度打击更有威胁。
    \item \textbf{两种策略均未击穿检测阈值}。即使在最激进的 20\% 随机替换条件下，Z=6.68 仍远高于 $\tau=4.0$。在当前参数配置（$\gamma=0.25$, $\delta=2.0$）下，基于词频的简单替换攻击无法使水印失效。
\end{enumerate}

\section{局限性与讨论}
这项工作存在若干局限，在此坦率列出：

\textbf{样本规模}。全部分析基于 10 组生成样本，总 token 量约 2,000。虽然 PPL 和 Z-score 的 Bootstrap CI 已经报告了不确定性范围，但 TVD 和 token 偏置排名的稳定性确实受到小样本的约束（TVD 的 Bootstrap CI 跨越 $[0.304, 0.403]$，区间宽度约 0.1）。扩大到 $\geq 50$ 组样本可能获得更稳定的偏置排序和更窄的 TVD 估计。

\textbf{模型与攻击器的泛化性}。本文仅限于 OPT-1.3B 单一模型和 T5 Paraphrase-PAWS 单一攻击器。不同模型族（LLaMA、GPT 系列等）的词表分布特征可能显著不同，本文的观测不应在未经检验的情况下外推。

\textbf{TVD 的序列化局限}。边际分布的 TVD 衡量的是逐 token 的独立偏置，但自回归生成中 token 选择受上文条件化支配。本文未在条件分布 $P(t_i \mid t_{1:i-1})$ 层面评估统计泄漏——那才是更贴近实际攻击场景的度量维度。

\textbf{攻击策略的简单性}。定向替换实验仅验证了基于词频的朴素攻击：攻击者不维护语义连贯性、不使用语言模型指导替换选择，也不构造等价表达。Sadasivan 等人~\cite{sadasivan2023detection} 指出，在足够的改写强度下，AI 生成文本可与人类写作在统计上不可区分；更精细的攻击（如利用 LLM 在语义约束下生成等价替换 token）可能取得更好的效果，但也会付出更高的攻击成本。

\section{结论}
本文对 Kirchenbauer 红绿名单水印方案进行了一次独立的复现和统计检验。数据确认了该方案的核心优势：高检测强度（Z-score 均值 10.04，检测率 100\%），以及对 T5 释义攻击的一定鲁棒性（攻击后 Z=5.24 仍高于阈值 $\tau=4.0$，检测率 90\%）。与此同时，TVD 和 KL 分析揭示了词表分布层面存在客观的统计偏移，偏移主要集中在少数高频功能词上。但利用这一偏移构造的定向替换攻击并未对该方案的检测可靠性构成实质威胁：在高替换率区间，攻击效果迅速进入平台期，两种替换策略均无法将 Z-score 压制到 4.0 以下。

本文不声称理论创新或方法改进。这些数据和观察的价值，在于为 Kirchenbauer 方案的统计行为提供了一份独立、透明、可复现的记录——所有数值均可通过项目代码完整回溯。文中揭示的分布偏置现象以及定向攻击的平台效应，也许能为后续更深入的密码学安全分析提供一点经验素材。
